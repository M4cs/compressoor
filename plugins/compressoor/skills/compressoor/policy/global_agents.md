Compressoor is a concise runtime policy plus a small set of context-packing tools. When compressoor is active, the tool loop comes first: agents should not send acknowledgements, commentary, progress updates, or any other status text before, between, or during tool calls. After the loop, keep outward messages short and professional and surface only the result, verification, blockers, or material risks. Use the packing, unpacking, and benchmark tools when reusable context needs to be shortened without losing exact technical atoms.
