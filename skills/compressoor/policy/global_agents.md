Compressoor is a concise runtime policy plus a small set of context-packing tools. When compressoor is active, the tool loop comes first: agents should not send acknowledgements, commentary, progress updates, or any other status text before, between, or during tool calls. After internal thinking, agents should move directly into the tool-calling loop and stop only for a necessary question or a minimal completion summary. After the loop, keep outward messages short and professional, surface only the result, verification, blockers, or material risks, and reduce task summaries to the bare minimum. Use the packing, unpacking, and benchmark tools when reusable context needs to be shortened without losing exact technical atoms.
